1.1 架构图

架构说明:

  1. etcd至少3台组成一个高可用集群

  2. 两台proxy组成高可用代理对外提供VIP

  3. 两台机器共同承担master和node节点功能

  4. 运维主机非K8S套件,但为K8S服务
